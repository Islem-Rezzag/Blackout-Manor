# Visual Language

## Core Statement
Blackout Manor should read as a premium 2D/2.5D cutaway manor thriller: Sims-like cutaway readability, storm-noir manor atmosphere, masquerade humans.

It is explicitly not Among Us visual identity.

## Identity Lock
- Not Among Us visual identity
- Not beans, capsules, or simplified widgets
- Not flat toy-room cards
- Not a sterile benchmark control room
- Yes to stylized realistic humans
- Yes to a connected gothic manor
- Yes to readable furniture, doors, corridors, and thresholds
- Yes to spectator-first cinematic clarity

## Camera Model
- Default: wide cutaway overview that lets a reviewer read multiple spaces at once
- Room focus: medium zoom that preserves context to adjacent doors and corridor entries
- Surveillance mode: multi-feed in-world console using the same room language
- Event override: short focus pulls for sabotage, report, meetings, and other major public actions

## Spatial Readability Rules
- Every room must have a clear entrance and exit
- Major rooms need one memorable anchor shape visible from the overview
- Corridors must visually explain travel between wings
- Thresholds should be obvious before a character crosses them
- Task hotspots should read as part of the room, not as floating UI markers
- Cutaway walls should hide obstruction, not erase room identity

## Room Readability Rules
- One hero prop cluster per room
- One readable path through the room
- One obvious interaction side for the main hotspot
- Furniture should support the fiction of the room, not only decorate empty space
- Each room needs a distinct silhouette when viewed from the whole-manor camera

## Character Silhouette Rules
- Distinguish characters by shoulders, coat shape, skirt or trouser cut, cape or shawl presence, mask profile, and accessory size
- Avoid relying only on color to differentiate agents
- Keep head-to-body proportions realistic enough to read as humans
- Preserve readability at the full-manor camera scale

## Color Language
- Warm amber, brass, and candle-gold for inhabited interior spaces
- Cold slate-blue, moon-gray, and rain-silver for windows, storm edges, and surveillance mood
- Deep claret, forest green, and muted jewel tones for masquerade accents
- Alert colors should feel like diegetic danger, not arcade neon

## Lighting Language
- Warm interior light versus cold blue storm exterior light is the global contrast rule
- Blackouts should feel sudden, dramatic, and spatial
- Lightning should briefly reveal silhouettes and reflective surfaces
- Surveillance should lean colder and flatter than direct room observation, but remain within the same world
- Meetings should gather warmer focal light around the table while preserving storm pressure in the surrounding shell

## Material Language
- Wood, brass, velvet, marble, glass, and rain-wet stone
- Avoid plastic, sci-fi panels, or overly clean modern surfaces
- Floors should subtly telegraph room function through material change

## Spectator UI Tone
- Quiet, premium, cinematic
- HUD supports the world; it does not compete with it
- Labels should feel like a broadcast overlay, not a dev inspector
- Benchmark and debug panels remain off the live route

## Portrait Direction
- Masquerade portrait framing
- Soft directional light
- Consistent identity markers between portrait and in-world avatar
- Slightly heightened theatricality during meetings, but not comic exaggeration

## Motion Tone
- Readable, intentional, slightly stage-like
- Characters should look like they decide to move, not slide
- Meetings should feel blocked like actors taking marks

## Definition Of Failure
- If the game looks like a dashboard with sprites
- If the house reads as disconnected room cards
- If characters read as tokens instead of people
- If surveillance looks like a separate website instead of an in-world console
